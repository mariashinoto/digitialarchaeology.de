8  Software

8.1 Mit Hardware verknüpft

  • Encoding: ASCII etc. -> Unicode (UTF8)
  • Verarbeitung von Schrift, Farben, Ton
  • Betriebssysteme kompatibler
  • Traditionelle Hardware
    • traditionellere Konzepte
    • weniger Integration
  • Mobile Devices
    • Auseinanderdriften der Anwendungen – vollkommene Austauschbarkeit mit traditioneller Software oder Entwicklung neuer Programme mit neuem Bedienkonzept?
    • Nutzen der Integrationsmöglichkeiten durch zusätzliche Ausstattung
    • Nutzen der Mobilität
    • (Ausgrabungsdokumentation Hodder…)

8.2 Programmgrundlagen

  • Konzepte bleiben
    • Module, Libraries, Handler, Methoden…
    • Entscheidungen nach Konditionen, Schleifen, …
    • Variablen und Konstanten
    • Pseudo-Code
    • Problemlösungen; Programmdesign
  • Frage: Was muss davon vermittelt werden?
  • Neu: Austausch, OpenSource, Dependencies, offene Wissensvermittlung

8.3 Dateiformate

  • Entwicklung von proprietär über proprietär und öffentlich dokumentiert bis zu offenen Standards –> extrem flexibel und integrierbar

8.4 Grundlegende Gruppen von Programmen und Daten

  • Textverarbeitung
  • Bild- (Photo-) Verarbeitung
  • Zeichnung
  • CAD
  • 3D vektorbasiert und pixelbasiert (auf Basis von Bild und Zeichnung)
  • Datenverarbeitung
    • Datenmodelle, SQL, XML, … (von traditionell zu heute und Zukunft)
  • Tonverarbeitung
  • Video als Kombination von Photo und Ton
  • GIS als Kombination von Zeichnung, Bild, Daten, (+ Code, +Geographie)
  • Sonstiges?

8.5 Software-“Klassiker” und moderne Alternativen

  • Text: Word -> Markdown + Pandoc (Quarto)
  • Photo / Bild: -> Zahlreiche Alternativen
    • TODO – GIMP als OpenSource und plattformübergreifend wenig erfolgreich
    • kostenpflichtige Programme mit proprietären Speicherformaten (für die Verarbeitung) und Export in Standardformate (TIFF, JPEG, HEIC etc.)
  • Zeichnung: Wie bei Bildern; SVG mit Einschränkungen für Vektor-, CAD, GIS
  • Zeichnung mit Datenbank und Massstab (Richtung GIS, ohne Geologie)

8.6 Integration klassisch getrennter Aufgaben

  • Seit langem: Bild und Text
    • auch: Medien und Text; also Video, Ton, etc.
  • “Literate Programming”: Code und Text
  • Texteditoren / Textverarbeitungsprogramme, die wie Code agieren
    • LaTEX
    • Markdown
    • Pandoc -> RMarkdown -> Quarto
    • Andere Gruppe: Jupyter Notebooks
